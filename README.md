# AI助手命令参考

## 设置上下文命令

```
@context 请严格按照/doc/开发规范.md 进行开发，每次修改前必须：
1. 检查是否符合规范
2. 更新 CHANGELOG.md
3. 说明修改依据
```

# 工作流指令

```
@workfLow
1. 查看相关文档
2. 评估修改方案
3. 执行代码修改
4. 更新 CHANGELOG.md
```

## 任务指令（完整版）

```
@task 任务描述
@docs
- /doc/开发规范.md
- /doc/CHANGELOG.md
- /doc/CHECKLIST.md
- /doc/数独游戏规则.md
- /doc/数独棋盘渲染规范.md
- /doc/数独棋盘状态管理规范.md

@requirement
请在执行任务前：
1. 检查相关文档
2. 确认符合规范
3. 说明修改依据
4. 更新必要文档
```

## 提交信息模板

```
类型: 简短描述

- 修改内容：
  1. 
  2. 

- 相关文档：
  - [ ] 开发规范.md
  - [ ] CHANGELOG.md

- 检查项：
  - [ ] 符合代码规范
  - [ ] 更新了文档
  - [ ] 添加了测试
```

## 使用说明

1. **开始新任务时**：
    - 复制任务指令模板（完整版或简化版）
    - 填写具体任务描述
    - 指定相关文档路径
    - 说明具体要求

2. **设置上下文时**：
    - 复制上下文命令
    - 可以根据需要修改检查项

3. **提交更改时**：
    - 使用提交信息模板
    - 填写具体修改内容
    - 检查相关文档更新
    - 确认所有检查项

4. **最佳实践**：
    - 总是指定具体的文档章节
    - 明确说明修改依据
    - 保持文档同步更新
    - 完成所有检查项
